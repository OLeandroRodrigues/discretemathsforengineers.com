Breadth-First Search (BFS)

Part 1 — Graph Model & Problem Setting

1. Graph Model

Breadth-First Search operates on a graph formally defined as

G = (V, E)
  • V — the set of vertices
  • E — the set of edges

Graphs may be:

  • Directed
  • Undirected

BFS works for both, but its interpretation may differ depending on edge direction.

2. The Search Problem

Given a graph G = (V,E) and a starting vertex s ∈ V, the goal is to explore all vertices reachable from s.

BFS systematically visits vertices in increasing order of distance from the source.

Key idea: vertices closer to the source are explored first.

3. Distance in Graphs

In unweighted graphs, the distance between two vertices is measured as the number of edges in the shortest path connecting them.

δ(s,v)

This represents the shortest-path distance from the source s to vertex v.

BFS computes these distances exactly for unweighted graphs.

4. Reachability

A vertex v is said to be reachable from s if there exists a path from s to v.

BFS discovers exactly the set of vertices reachable from the source.

This property makes BFS useful for connectivity analysis and network traversal.

5. Why BFS Matters

BFS is one of the most fundamental graph traversal algorithms.

It serves as the foundation for many important problems in computer science.

  • Shortest paths in unweighted graphs
  • Network broadcasting
  • Minimum number of moves in puzzles
  • Social network distance analysis
